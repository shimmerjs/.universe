Rows 1-8 are the detail zone (fixed 8 rows while occupied, so the list never shifts under a finger), read as one card behind the left rail: every zone row -- blank pads included -- leads with a thin bar hued to the occupant's identity (the same hue as its name in the list), so the zone's extent and WHOSE detail it is are both visible at a glance. Row 1: header = the session's list row + model and effort appended dim. Row 2: the outcome line -- here the needs-you branch: warn age (time since the gate), typed glyph, notification text (falls back to the registry's waitingFor, then "input requested"). Rows carry the amber wash because this session is registry-waiting. Note its age tone is dim: blocked is not working -- the wash marks need, tone marks activity. "no agents" fills the reserved fleet space so it doesn't read as a dead gap.

Rows 9-13, the list zone: row 10 busy + fleet: bright 10s age, blue ag 3 / wf 1 live-fleet counts (blank at zero elsewhere). row 11 the codex-consult case: 4m-stale files but registry-busy, so the age is STILL bright -- and blank state glyph = turn in flight. Name falls to cwd basename "can" (no title yet, derived registry names never display). row 12 idle: dim, check = turn done. row 13 idle after a failed turn: cross = StopFailure. Name hues are keyed to the session id (stable across renames); cwd column is fish-abbreviated repo-relative ("can-work/containers/can" -> "c/c/can"). Every session row is a tap target: focus its kitty window.

Scenario: busy session with a fleet (accordion expanded)

▏ 10s ag 3 wf 1 universe panel-work > fix clod panel attention semantics claude-fable-5 xhigh
▏ 5m turn running
▏ 20s v wf aw-review 2/4
▏ ├─20s ag reviewer review:security:r1
▏ ├─40s ag skeptic review:codex:r1
▏ ├─ 1m ag researcher verify:panel.go
▏ └─ 1m ag mapper synthesize
▏ +2 trees

Outcome line shows the in-flight branch: accent age since the prompt + "turn running". The fleet is a greedy accordion: the leading tree expands vertically; trees that lost the space race collapse into "+2 trees". Tapping ANY row of a tree toggles that tree's fold. Name-plates (new): the root says aw-review -- the run name read from any child's prompt plate, instead of the opaque wf_7fc5bc03-3c4 dir name -- and each child's description column is its plated leg name (review:codex:r1, verify:panel.go), so you can tell the cross-model leg from a lens reviewer from a verifier at a glance. Running children bright, finished dim.

Scenario: same fleet, leading tree folded

▏ 10s ag 3 wf 1 universe panel-work > fix clod panel... claude-fable-5 xhigh
▏ 5m turn running
▏ 20s > wf aw-review 2/4 1 mapper 1 researcher 1 reviewer 1 skeptic
▏ 30s v ag agents 1/2
▏ ├─30s ag general-purpose Sweep panel_test.go fixtures
▏ └─ 3m ag claude-code-guide Registry status enum research
▏ 20s v wf khudson-ledger-map 0/3 1 researcher 1 reviewer 1 skeptic
▏ └─ +3 agents

Folding aw-review (closed chevron >) collapses it to a one-line summary -- the type breakdown ("1 mapper 1 researcher...") appears exactly when children are hidden, so the info never vanishes with the rows. Its freed rows flow to the NEXT tree: agents (loose Agent-tool spawns, which carry their real descriptions from meta -- "Sweep panel_test.go fixtures") now shows children; khudson-ledger-map (an inline workflow, named from its plate) gets a root with inline summary and a "+3 agents" stub. 0/3 live = fully dim tree. The rail runs the full zone in both scenarios -- fold state changes rows, never the card.

Overflow + empty states

+3 more
no active sessions
no session in focus

"+N more" caps the list zone (not tappable). "no active sessions" = zero live Claude processes (or, pathologically, a registry fault would now surface as a widget ERROR, never this). "no session in focus" = the detail zone's placeholder in the zero/one-idle states where the zone collapses.
