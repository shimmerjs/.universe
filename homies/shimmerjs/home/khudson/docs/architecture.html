khudson architecture

as deployed 2026-07-15 (aw-chainguard). Boxes with a launchd label are KeepAlive LaunchAgents; IM AX SR mark macOS TCC grants (Input Monitoring, Accessibility, Screen Recording); ID marks a binary signed with the persistent shimmerjs identity at a fixed out-of-store path -- the pair the grant keys on.

signing and grant continuity

  • Both TCC clients are copied out of the store to fixed paths and signed with the persistent login-keychain identity shimmerjs (per-host option, single source for both binaries). TCC keys on path + signature: ad-hoc signing would rotate the cdhash every rebuild and silently kill the grant; a store path would re-prompt per generation.
  • No hardened runtime: library validation rejects nix-store dylibs ("different Team IDs") and the daemon dies in dyld. Grants key on identity, not hardened runtime.
  • Staging signs beside the granted binary and replaces atomically only after codesign succeeds, so a missing/expired cert can never leave an unsigned copy where the granted binary was.
  • Login Items (BTM) name agents by ProgramArguments[0] basename -- every agent execs a named launcher from App Support/khudson/agents/, so the list reads magicbusd, hud, bus, substrate instead of sh.
  • Binary/label renames are a new TCC subject (the magicbusd rename cost one accepted Input Monitoring re-grant); an activation leg migrates the old agent out.

today vs post-swap input

The third-party gestures driver still owns the digitizer: taps reach the dock as kitty mouse clicks and a touch long-press arrives as the driver's hold-as-right-click, which the dock routes to the same context-menu opener as a recognizer LongPress. magicbusd's edge module stays in seized backoff (bounded, 5min cap) until the driver is retired -- the parked swap decision. After the swap, touch.sock feeds the bus recognizer and the full gesture vocabulary (drag, flick, two-finger) replaces the driver's click vocabulary; nothing else in the diagram moves.
